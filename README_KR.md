# 스킨케어 AI 에이전트 — 재고 최적화 및 개인화 추천 시스템

> **POC 프로젝트** 

화장품 리테일의 두 가지 비즈니스 문제를 동시에 해결하는 멀티 에이전트 AI 시스템입니다:
- **재고 정체** — 신제품 빠른 출시로 인한 재고 회전율 저하
- **고객 이탈** — 개인화되지 않은 일반적인 마케팅으로 인한 낮은 재구매율

자연어 기반 피부 분석과 실시간 재고 현황을 결합하여, 고객-제품 1:1 매칭을 스케일로 구현합니다.

---

## 문제 정의

| 이해관계자 | 핵심 페인 포인트 |
|------------|----------------|
| 브랜드 (아모레퍼시픽) | 잦은 신제품 출시로 인한 재고 적체, 재고 회전율 저하 |
| 플랫폼 | 획일적 추천 로직 → 낮은 클릭률 및 재구매율 |
| 고객 | 개인 피부 상태를 반영하지 않는 추천 |

---

## 시스템 아키텍처

CrewAI 기반 **3-에이전트 순차 파이프라인**:

```
사용자 자연어 입력
        ↓
┌───────────────────────┐
│ 피부 과학 및 성분 분석가 │  → 피부 타입·고민·민감도·환경 요인 추출
│  (Skin Analyst)        │    → 맞춤 성분 가이드라인 도출
└───────────────────────┘
        ↓
┌──────────────────────────┐
│ 재고 최적화 및 제품 매칭 전문가 │  → amore_inventory.csv 조회
│  (Inventory Manager)        │    → 효능 기반 매칭 + 재고 우선순위 적용
└──────────────────────────┘
        ↓
┌──────────────────────────┐
│ 뷰티 콘텐츠 마케터          │  → 고객 맞춤형 추천 리포트 생성
│  (Marketing Specialist)   │    → 공감형 문체, 한국어 출력
└──────────────────────────┘
        ↓
  개인화된 추천 리포트
```

### 핵심 설계 원칙

- **재고 가중치 매칭** — 효능이 유사한 제품 중 재고 충분 제품을 우선 추천, 재고 회전율 KPI에 직접 기여
- **데이터 프라이버시 보장** — 에이전트 간 중간 추론 과정 및 원본 CSV 데이터가 최종 출력에 노출되지 않도록 태스크 레벨에서 명시적 제어
- **자연어 인터페이스** — *"요즘 피부가 푸석해"* 같은 구어체 입력을 처리 (폼/드롭다운 불필요)

## 빠른 시작

### 1. 클론 및 의존성 설치

```bash
git clone https://github.com/<your-username>/skincare-ai-agent-poc.git
cd skincare-ai-agent-poc

pip install crewai crewai-tools pandas python-dotenv openai
```

### 2. API 키 설정

```bash
cp .env.example .env
# .env 파일을 열고 OpenAI API 키를 입력하세요:
# OPENAI_API_KEY=sk-proj-...
```

### 3. 노트북 실행

Jupyter에서 `notebooks/recommendation_agent.ipynb`를 열고 전체 셀을 실행합니다.

```python
# 사용 예시
user_request = "나는 건성에 성분에 민감한 타입이고 요즘 피부가 푸석해"
response = get_realtime_recommendation(user_request)
print(response)
```

**출력 예시:**
```
✨ 아모레퍼시픽 AI 카운셀러의 응답:
- 제품명: 수퍼 모이스처라이징 크림
- 고객 피부 타입 및 적합 성분: 건조하며 푸석함과 민감성으로 고민이시군요.
  히알루론산, 글리세린, 세라마이드 성분이 수분 보존과 보호막 강화에 도움을 줍니다.
- 해당 제품의 효과: 지속 사용 시 건조함·푸석함이 개선되고
  촉촉하고 탄력 있는 피부를 경험하실 수 있습니다.
```

## 재고 데이터 스키마

| `타입효과` | 주요 피부 효과 |

## 향후 개선 방향

- **벡터 DB 연동** — Qdrant를 활용한 시맨틱 유사도 검색 도입: *"푸석하고 당기는"* 같은 추상적 표현을 제품 성분 임베딩과 매칭
- **배치 처리** — `users_100_example.csv` 전체 고객 대상 CrewAI 배치 모드로 B2B 캠페인 대량 생성
- **클라우드 배포** — AWS Lambda (에이전트 실행) + S3 (재고 데이터) + API Gateway (고객 엔드포인트)
- **실시간 재고 연동** — 정적 CSV 대신 실시간 재고 API 연결
